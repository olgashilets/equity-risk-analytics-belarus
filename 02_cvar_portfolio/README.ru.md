# 02. CVaR-оптимизация портфеля

[English version](README.md)

## Назначение

Кейс строит защитный long-only портфель через исторический CVaR / Expected Shortfall на уровне доверия 95%.

Проект специально сфокусирован только на портфельной оптимизации. Инвестиционная вселенная **заранее задана** и загружается из `data/input/selected_asset_prices.csv`. Кластеризация внутри этого модуля не повторяется.

## Бизнес-вопрос

Может ли CVaR-оптимизированный портфель снизить ожидаемые потери в худших исторических месячных сценариях по сравнению с равновесным портфелем и портфелем минимальной волатильности?

## Входные данные

```text
data/input/selected_assets.csv
data/input/selected_asset_prices.csv
```

Выбранная инвестиционная вселенная получена из предыдущего кейса по кластеризации. Такое разделение делает проект чище: входной набор активов фиксирован, а модуль занимается только портфельной аналитикой.

## Методология

- Очистка выбранных месячных цен.
- Расчёт месячных логарифмических доходностей.
- Построение трёх портфелей:
  - Equal Weight;
  - Minimum Volatility;
  - CVaR 95% / Expected Shortfall.
- Сравнение годовой доходности, годовой волатильности, VaR 95%, CVaR 95%, максимальной просадки и Sharpe ratio.

## Как запустить

Из папки проекта:

```bash
python run_cvar_portfolio.py
```

## Ограничения

Выборка короткая, рынок малоликвидный. Исторический CVaR зависит от наблюдавшихся сценариев и должен проверяться out-of-sample перед практическим использованием. Проект является аналитическим кейсом и не является инвестиционной рекомендацией.
